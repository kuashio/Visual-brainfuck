Basic Controls

Visual brainfuck has three text editors:

        • Input. The text you want your program to take as input with the , instruction.
        • Program. Your brainfuck code with syntax highlight.
        • Output. The output sent with the . instruction.


The 3 basic text editors in Visual brainfuck


The Program Editor

Syntax highlight prints the brainfuck characters (<>,.+-[]) in yellow, the Visual brainfuck breakpoint character (~) in red, and all the rest in gray, since everything other than tie brainfuck characters is considered a comment.


One cool feature is the ability to comment actual brainfuck code out, by placing it between "[-][{" and "}][-]". Brackets MUST be balanced in the commented code, though.


The Input Editor

Whenever a , instruction is executed, the Input Editor will lose its first character as it will be taken into the pointed cell.


If a , instruction is executed and there's nothing in the Input Editor, an input dialog box will pop up asking for the input. This dialog box accepts the same as the Input Editor: not only one character, but a whole string.

The Input Dialog Box


Both the Input Editor and Dialog Box have an "Enter Char" button that allows the user to enter values that aren't easy to enter in ASCII (for example the number 22). It also accepts the following escape characters as their respective values:
